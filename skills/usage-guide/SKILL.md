# Session Semantic Search

## 核心价值

过往对话的语义检索。对话的价值不在标题里，而在对话过程之中。用户可能记不清当时用了什么词、什么表述，但能描述"大概是什么事"——这正是语义搜索擅长的。

## 何时使用

### 显式触发（用户明确要求查历史）

当用户说出类似以下的话时，必须调用 search：

- "之前我们讨论过 XXX 相关的内容"
- "我记得你说过关于 XXX 的事，帮我找找"
- "我们是不是聊过 XXX 话题"
- "上次那个关于 XXX 的方案/想法/代码"
- "你之前分析过 XXX 的问题对吧"

### 隐式触发（语义推断——核心升级）

**不要只等用户说"搜一下"。** 以下场景即使没有显式回溯词句，也应主动调一次 search，用当前讨论的主题构造 query：

| 场景 | 判断逻辑 | 示例 query 构造 |
|------|---------|---------------|
| **技术决策/方案评估** | 用户在选方案、对比工具、问"要不要用XXX" | "对 [工具/方案] 的评估和讨论" |
| **问题诊断/bug 修复** | 用户描述一个异常现象、报错、奇怪行为 | "[技术栈/错误现象] 相关的错误和修复" |
| **工具/库选型** | 用户提到一个没接触过的工具或库 | "对 [工具名] 的试用和评价" |
| **代码/架构讨论** | 用户在审视某段代码或结构，准备改动 | "[模块/项目] 的架构设计和讨论历史" |
| **概念回顾** | 用户问一个技术概念、原理、区别 | "关于 [概念] 的解释和讨论" |
| **配置/环境问题** | 用户在折腾环境、配置、部署 | "[技术栈] 的配置和部署问题" |

**核心原则**：不纠结"用户有没有说"，而是想"当前这件事，历史里有没有相关记录"。只要有合理的可能性，就花一次 embedding 的代价去查。

### 什么时候不需要搜

- 纯闲聊、打招呼、情感交流
- 用户明确在说一件全新的事且说了"我不确定以前有没有聊过"以外的话
- 当前问题完全不需要参照历史的场景（如"现在几点""帮我写一封邮件"）

## 使用流程

### 1. 检查服务状态
```
search action=health
```
确认嵌入服务（Ollama）在线。如果不在线，提醒用户启动 Ollama。

### 2. 检查索引状态
```
search action=stats
```
看索引中有多少会话和片段。如果为空（totalChunks=0），先做一次全量索引。

### 3. 索引管理
- **自动机制**：插件每 30 分钟自动扫描，支持**增量追加**更新，并能自动检测会话的结构性变动（如被手动改写），触发全量自愈。
- **强制重建**：如果索引状态明显异常（如内容明显缺失），使用：
```
search action=reindex
```

### 4. 语义搜索（核心功能）
```
search action=search query="用户的自然语言描述" [maxResults=10] [minSimilarity=0.5]
```

查询参数说明：
- `query`：自然语言描述，不要用关键词拼凑。越接近真实表达的句子效果越好。例如：
  - ✅ "关于 MCP 服务器安全配置的讨论"
  - ✅ "之前分析过 embedding 模型的优缺点"
  - ❌ "MCP 安全"（过于碎片化的关键词）
- `maxResults`：返回结果数（1-30，默认 10）
- `minSimilarity`：相似度阈值（0-1，默认 0.5）。阈值越低召回越多但噪音越大

> **注意**：search 工具默认返回 JSON 格式的结构化数据，适合 AI 解析。如需人类可读报告，传 `format=text`。

### 5. 结果解析策略

搜索结果以 JSON 格式返回，每条包含 score、sessionId、preview、topics、exchangeRange 等字段。按以下策略使用：

1. **高匹配（score ≥ 0.7）**：直接引用，把预览片段融入回答
2. **中匹配（0.5 ≤ score < 0.7）**：作为参考，用"我记得之前讨论过类似的问题……"的口吻引入
3. **低匹配（score < 0.5）**：通常不提及，除非内容特别相关
4. **无结果**：正常做你的分析，不提及搜索行为

## 技术说明

- **智能索引引擎**：
  - **原子问答切分**：不拆散问答对，保证语义连续性
  - **智能增量追加**：快速度追加新对话，低 IO 开销
  - **结构变动自愈**：检测到会话被编辑/重写后自动重建
- **嵌入模型**：默认使用 Ollama 本地 `qwen3-embedding:0.6b`
- **搜索算法**：暴力余弦相似度，万级片段量级性能充足
- **存储**：纯 JSON 文件，无原生模块依赖

## 注意事项

- 嵌入服务首次调用需要加载模型到内存，第一请求可能较慢
- 不要在 health 或 stats 返回异常时频繁重试，等几秒再试
- 如果结果不理想，优先换 query 表述而非调低 threshold
