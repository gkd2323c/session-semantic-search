# Session Semantic Search

会话语义搜索引擎——让过往对话真正变成可随时翻阅的知识库。

## 解决的问题

用了一段时间，session 越攒越多。很多对话的价值不在标题里，而在对话过程中。想找之前某次讨论过的东西，标题搜不到，关键词搜也不一定准——因为当时可能用了完全不同的表述。

## 能力

- **语义检索**：用自然语言描述要找的内容，不必精确回忆当时的用词
- **主动触发**：不仅用户说"查一下"才搜——技术决策、方案评估、问题诊断等场景，Agent 会自己判断需要查历史并主动搜索
- **上下文预览**：命中结果展示对话片段的前几轮对话，方便快速判断是否要找的内容
- **即时索引**：对话结束后数秒内自动索引新内容，不等定时扫描
- **智能索引**：
    - **原子问答切分**：确保上下文边界平滑，不再拆散用户提问与 AI 回答
    - **智能增量机制**：支持对会话进行增量追加，并能自动识别"重新编辑/重写"等结构性变动，触发全量自愈更新
    - **极低开销**：显著降低 IO 负担，适合高频对话环境
- **灵活配置**：可切换嵌入模型、调整分片策略和搜索参数
- **零原生依赖**：纯 JS 实现，无 native 模块，跨 Node.js 版本兼容

## 前置要求

需要运行一个嵌入模型服务。默认使用 [Ollama](https://ollama.ai/) 本地服务：

```bash
# 安装 Ollama 后拉取嵌入模型
ollama pull qwen3-embedding:0.6b
```

也支持其他兼容的嵌入 API（OpenAI `/v1/embeddings` 格式），在插件设置中修改 `embeddingEndpoint` 即可。

## 安装

1. 将 `session-semantic-search` 文件夹拖入 Hanako → 设置 → 插件页面
2. 或在设置 → 插件页面中通过文件选择器选中此文件夹
3. 确保已开启"允许全权插件"开关

## 使用方法

插件安装后会自动在后台索引历史对话。你可以直接在对话中自然地向 Agent 提问：

> "之前我们讨论过 MCP 安全相关的内容吗？"
> "我记得好像聊过关于 embedding 模型选择的事"
> "帮我找一下之前分析过的那个方案"

Agent 会自动调用语义搜索工具。此外，技术决策、配置排查、方案评估等场景下 Agent 也会主动搜索历史。

也可以手动使用工具（通过工具调用界面或显式指示 Agent）：

| 操作 | 说明 |
|------|------|
| `search action=health` | 检查嵌入服务状态 |
| `search action=stats` | 查看索引统计 |
| `search action=reindex` | 全量重新索引 |
| `search action=search query="..."` | 语义搜索 |

## 索引管理

- **即时索引**：每轮对话结束后数秒内自动索引，不等定时扫描
- **定时增量**：每 30 分钟自动检查并增量索引未覆盖的新会话
- **首次使用**：插件启动后自动执行全量索引，根据会话数量可能需要 1-5 分钟
- **手动强制**：如果索引状态异常，用 `search action=reindex` 重建

## 搜索技巧

1. **用句子描述，不用关键词**：`"聊过关于 MCP 服务器安全配置的事"` 比 `"MCP 安全"` 效果好得多
2. **换角度重试**：第一次没找到？换个说法试试，语义搜索对不同的表述方式敏感度不同
3. **调整阈值**：默认最低相似度 0.5，搜索结果偏少时可以降低到 0.3 展开召回

## 配置项

在插件设置页面可配置：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `embeddingEndpoint` | `http://localhost:11434/api/embed` | 嵌入 API 端点 |
| `embeddingModel` | `qwen3-embedding:0.6b` | 嵌入模型名称 |
| `chunkSize` | `10` | 每个片段包含的最大对话轮数 |
| `chunkOverlap` | `2` | 相邻片段重叠轮数 |
| `maxResults` | `10` | 搜索返回最大结果数 |
| `minSimilarity` | `0.5` | 最小相似度阈值 |
| `autoIndex` | `true` | 是否自动索引 |
| `indexIntervalMinutes` | `30` | 增量索引间隔（分钟） |
| `instantIndexDebounceMs` | `3000` | 即时索引防抖等待时间（毫秒） |

## 技术架构

```
用户查询 → 向量化 → 余弦相似度 → Top-N 片段 → 上下文预览
                ↑                      ↑
   Ollama qwen3-embedding       向量索引 (JSON)
```

- **嵌入**：Ollama `/api/embed` 端点，本地运行，不发送数据到外部
- **分片**：原子化问答滑动窗口，保证上下文语义完整
- **存储**：JSON 文件持久化，无原生模块依赖
- **搜索**：暴力余弦相似度（适合万级以下片段量级）
- **状态共享**：插件实例与搜索工具复用同一 Embedder 和 VectorStore，避免重复加载
